Zewijuna is a real matchmaking product delivered across a mobile application and supporting web platform.

About the Product

The product combines account and onboarding flows with profile discovery, preference-based filtering, realtime communication, membership logic, and administrative verification.

Kirubel worked across product flows, frontend implementation, Supabase-backed data, application logic, build configuration, and deployment work.

From onboarding to matching,
built as a connected system

Zewijuna was treated as an application, not a landing-page exercise. Authentication, profile setup, discovery, messaging, membership, and verification all depend on connected product state and database-backed workflows.

Product Experience

The mobile experience uses focused onboarding, card-based discovery, profile controls, filters, messaging, and membership screens designed for a clear progression from account creation to meaningful interaction.

The supporting web presence introduces the product and provides a polished route into the application.

Systems and Responsibilities

Responsibilities included product development, frontend flows, Supabase integration, database-backed features, realtime systems, membership logic, and application build and deployment work.

Verified product areas include phone authentication, onboarding, user profiles, discover, gender-based filtering, realtime chat, membership flows, and admin verification.

Technical Foundation

The mobile application uses Expo and React Native with Supabase. The project also includes backend routes, database schema work, payment and membership flows, notifications, and production build configuration.

Current State

Zewijuna exists as a working product codebase with mobile, web, backend, database, and deployment layers. No unsupported user, revenue, or conversion claims are presented here.

A complete product build

The work demonstrates Kirubel’s ability to move beyond interface design and build the connected systems a real application requires.
