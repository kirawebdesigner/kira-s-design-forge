# Design System and UI/UX Guidelines

## Design Principle

The exported Framer template is the visual and interaction source of truth. The transformation should preserve its premium editorial composition and adapt content to Kirubel without flattening the experience.

## Existing Visual Language

- Strong editorial typography
- Oversized display headings
- High-contrast neutral palette
- Structured grids with generous whitespace
- Carefully paced section transitions
- Image-led project cards
- Smooth scroll behavior
- Hover scaling and reveal effects
- Responsive variants generated by Framer

## Typography

Existing loaded families include:

- Archivo — primary body and interface typography
- Clash Grotesk — expressive display typography
- Inter — fallback and selected utility styles

Retain the existing hierarchy and responsive type scaling. Copy must be edited to fit the available visual measure instead of shrinking typography indiscriminately.

## Color

Retain the existing Framer color tokens and high-contrast neutral foundation during the first transformation pass. Exact token values will be inventoried from the generated CSS before custom overrides are introduced.

Do not introduce:

- Generic blue-purple AI gradients
- Excessive glassmorphism
- Repetitive glowing borders
- A generic rounded-card/bento system

## Layout

- Preserve current max widths, grid rhythm, section spacing, and alignment.
- Maintain project image aspect ratios.
- Adapt surrounding composition intelligently if template-only sections are removed.
- Mobile layouts must be intentionally composed, not merely compressed.

## Motion and Interaction

- Preserve Framer's existing page transitions, scroll behavior, hover states, image scaling, and menu interactions.
- Remove animation only if it is broken, inaccessible, or tied exclusively to removed template promotion.
- Respect `prefers-reduced-motion` behavior already provided by the export.

## Image Treatment

- Use real product screenshots only.
- Keep screenshots sharp, uncropped in misleading ways, and free of sensitive data.
- Use consistent framing per project type:
  - Mobile frames for Zewijuna mobile screens
  - Desktop/application frames for management dashboards
  - Telegram UI only where it accurately demonstrates YourCloser
- Do not stretch images to match containers; adapt `object-fit`, container ratio, or composition.

## Responsive QA Widths

320, 360, 375, 390, 412, 430, 768, 1024, 1440, and 1920 pixels.

Check typography, navigation, menu, cards, image crops, buttons, contact section, footer, animations, and overflow at each width.
