# AI Coding Guardrails

## Product Rules

- Preserve the exported template's design and interaction quality.
- Do not redesign from scratch, simplify the layout, or replace it with a generic developer portfolio.
- Publish only Zewijuna, YourCloser, DMS, Pommy, and Majestic.
- Treat the supplied clinic administration screenshot as DMS evidence.
- Write concise, specific, human copy focused on products, systems, and operational software.
- Avoid generic portfolio filler and unsupported claims.

## Evidence Rules

- Do not invent project facts, technologies, outcomes, metrics, links, testimonials, clients, or screenshots.
- Verify facts from the objective, local sources, documentation, real application behavior, or confirmed deployments.
- When evidence is incomplete, use restrained temporary copy and track the missing information.

## Screenshot Rules

- Capture screenshots from the real product whenever accessible.
- Do not use stock images, generated product mockups, unrelated interfaces, or recreated screenshots.
- Review every screenshot for secrets, personal data, private conversations, phone numbers, customer data, and broken states.
- Preserve aspect ratios and use consistent presentation.
- Use placeholders only after reasonable local, repository, URL, and startup investigation fails.

## Code and File Rules

- Treat generated Framer CSS, runtime, breakpoints, and interaction bindings as sensitive.
- Make the smallest reliable changes that preserve behavior.
- Synchronize repeated global content across all retained pages.
- Use local relative paths for new portfolio assets.
- Do not add dependencies without a clear need.
- Do not initialize, push, or modify a Git remote without explicit instruction.
- Do not delete surplus pages until inbound and outbound links have been remapped.

## Error Handling

- Fail visibly during transformation scripts when an expected source pattern is missing or appears an unexpected number of times.
- Keep backups through deliberate patches or generated diffs; never use destructive repository reset commands.
- Report unavailable tools and checks accurately.

## Accessibility

- Preserve semantic headings and landmarks where present.
- Add meaningful alt text to real screenshots.
- Keep keyboard navigation and visible focus behavior functional.
- Preserve reduced-motion support.
- Maintain readable contrast and avoid text embedded only in images.

## Validation Rules

- Serve through HTTP for browser testing.
- Test every required viewport.
- Check for horizontal overflow, clipped headings, broken cards, image distortion, navigation failures, and console errors.
- Re-scan all retained HTML for template identity, fake links, and excluded-project references before completion.
